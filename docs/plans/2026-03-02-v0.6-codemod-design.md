# zodvex v0.6 Migration Tooling — Design

**Date:** 2026-03-02
**Status:** Approved

## Context

zodvex v0.6 introduces breaking renames (Codec* → Zodvex* prefix) and deprecates several APIs in favor of `initZodvex`, `defineZodModel`, and `defineZodSchema`. Consumers upgrading from pre-0.5/pre-0.6 need migration assistance across three tiers of complexity:

1. **Simple renames** — unambiguous identifier swaps (automatable)
2. **Function call renames** — `zid()` → `zx.id()` (automatable with regex)
3. **Structural rewrites** — `zodTable` → `defineZodModel`, builders → `initZodvex`, `convexCodec` → `decodeDoc`/`encodeDoc` (require understanding project setup, not safely automatable)

## Deliverables

### 1. Automated Codemod — `bin/migrate-0.6`

A bash script runnable via `npx zodvex migrate ./convex` or `bunx zodvex migrate ./convex`.

**Renames (global identifier replacement):**

| Old | New |
|-----|-----|
| `CodecDatabaseReader` | `ZodvexDatabaseReader` |
| `CodecDatabaseWriter` | `ZodvexDatabaseWriter` |
| `CodecQueryChain` | `ZodvexQueryChain` |
| `CodecRules` | `ZodvexRules` |
| `CodecRulesConfig` | `ZodvexRulesConfig` |
| `createCodecCustomization` | `createZodvexCustomization` |
| `createCodecHelpers` | `createBoundaryHelpers` |
| `CodecHelpersOptions` | `BoundaryHelpersOptions` |

**Function call transform:**
- `zid('tableName')` → `zx.id('tableName')`
- Updates import specifiers accordingly (remove `zid`, add `zx` if not present)

**Behavior:**
- Targets `.ts` and `.tsx` files in the specified directory
- Skips `node_modules/`, `.git/`, non-TypeScript files
- Supports `--dry-run` flag (prints changes without modifying files)
- Prints summary of changes made and remaining deprecated API usage found

**Does NOT touch:**
- `zodTable` → `defineZodModel` (structural)
- `zQueryBuilder` etc. → `initZodvex` (structural)
- `convexCodec` → `decodeDoc`/`encodeDoc` (structural)
- `mapDateFieldToNumber` → `zx.date()` (different API shape)

**Packaging:** `bin` entry in `packages/zodvex/package.json`:
```json
{ "bin": { "zodvex": "bin/migrate-0.6" } }
```

### 2. Migration Guide — `docs/migration/v0.6.md`

Written guide covering structural changes the codemod can't handle:

**Section A: zodTable → defineZodModel + defineZodSchema**
- Extract model definitions into client-safe file(s)
- Replace `zodTable('name', shape)` with `defineZodModel('name', shape)`
- Replace `defineSchema({ table: Model.table })` with `defineZodSchema({ table: Model })`
- Note property changes: `.shape` → `.fields`, `.zDoc` gone (use `.schema.doc`), top-level `.docArray` gone (use `.schema.docArray`)
- Mention new chainable `.index()` / `.searchIndex()` / `.vectorIndex()`

**Section B: Builder → initZodvex**
- Consolidate scattered builder calls into single `initZodvex(schema, { query, mutation, action, ... })`
- Replace `zCustomQueryBuilder(query, customization)` with `zq.withContext(customization)`
- Show recommended `convex/zodvex.ts` setup file pattern

**Section C: convexCodec → decodeDoc/encodeDoc**
- `convexCodec(schema).encode(value)` → `encodeDoc(schema, value)`
- `convexCodec(schema).decode(value)` → `decodeDoc(schema, value)`
- `.pick(keys)` has no direct equivalent; use `safePick` + `encodeDoc`/`decodeDoc`
- Note: `initZodvex` DB wrappers handle this automatically

**Section D: mapDateFieldToNumber → zx.date()**
- Replace `z.date()` with `zx.date()` in schemas, remove `mapDateFieldToNumber` calls

**Section E: Removed/hidden internals**
- List of symbols that were public pre-0.6 but are now internal: `customFnBuilder`, `registryHelpers`, `makeUnion`, `pick`, `formatZodIssues`, `handleZodValidationError`, `validateReturns`, `assertNoNativeZodDate`, `isZodUnion`, `getUnionOptions`, `assertUnionOptions`, `createUnionFromOptions`

### 3. Agent Skill — `docs/skills/migrate-to-v0.6.md`

A Claude Code skill consumers can add to `.claude/skills/` for interactive migration.

**Trigger:** "migrate to zodvex 0.6", "upgrade zodvex", or similar.

**Process (rigid, fixed sequence):**
1. Run `bin/migrate-0.6` to handle automated renames
2. Scan for remaining deprecated API usage via grep
3. For each deprecated pattern found, walk user through the structural migration:
   - Show the specific file and usage
   - Present the before/after transform
   - Apply after confirmation
4. Verify: run `tsc --noEmit` after each file to catch regressions
5. Final check: grep for remaining deprecated imports

**Migration order:** models → schema → builders → codec → cleanup

The skill references the migration guide for context and lives in zodvex's repo so it stays in sync with the API.

## Consumer Discovery Flow

1. Upgrade `zodvex` to 0.6 in `package.json`
2. TypeScript errors and `@deprecated` warnings surface in IDE
3. `npx zodvex migrate ./convex` handles automated renames
4. Remaining deprecation warnings point to `docs/migration/v0.6.md`
5. If using Claude Code, the skill handles structural migrations interactively
